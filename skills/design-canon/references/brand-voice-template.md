# Brand voice template

> Шаблон файла `BRAND-VOICE.md` для дизайн-системы. Заполняй для конкретного продукта.

```markdown
# Brand voice — {Product name}

> Документ описывает голос продукта в текстах: UI, маркетинг, ошибки, документация. Все text-elements должны соответствовать.

> Версия v1. Обновляется через явный update + commit.

## 3. Третье лицо или «вы»?

- **Третье лицо:** «Платформа делает X» / «Система ведёт учёт Y»
- **Уважительное "Вы":** «Вы можете…» / «Если у Вас вопросы»
- **Casual "ты":** «Если ты…» / «Сделай X» (только для casual-продуктов)

## 4. I-voice / We-voice

- Используем ли «мы» в маркетинге? {да/нет}
- Используем ли «я/мы» в UI? {обычно нет}

## 5. Casing для заголовков

- **Sentence case:** «Что получает бизнес» (предпочтительно для B2B)
- **Title Case:** «Что Получает Бизнес» (US-style, реже для русского)

Выбор: {___}

## 6. Кавычки и punctuation

- **Кавычки для названий:** «ёлочки» (русский) / "english double" (английский) / 'single' (никогда не использовать с другими)
- **Em-dash style:** with spaces (— как в этом тексте) / without spaces (—как-то-так)
- **Multiple punctuation:** запрещено (нет «!!!» или «???»)

## 7. Запретные слова / фразы

- {Слово 1} — {почему}
- {Слово 2}

Например для conservative B2B:
- «Революционный», «уникальный», «лучший», «№1» — нет суперлативов
- Восклицательные знаки — нет
- Эмодзи в running copy — нет
- «Кликни», «залогинься» — нет англицизмов-как-глаголов
- «Просто как 1-2-3» — нет marketing-pop фраз

## 8. Предпочтительные слова / фразы

(brand vocabulary — слова которые отражают позиционирование)

- {Слово 1} — {когда используется}
- {Слово 2}

## 9. Числа

- **Разделитель разрядов:** thin space (4 750 000) / запятая (4,750,000) / точка (4.750.000)
- **Decimal separator:** запятая (21,4%) / точка (21.4%)
- **Currency:** ₽ через NBSP / $ перед числом / € через NBSP / etc.
- **Negative:** −12 000 ₽ (с настоящим minus character `−`, не hyphen `-`)
- **Percent:** 21,4% (без пробела перед %) или 21,4 % (с пробелом — чаще для французского)

Выбор формата: {___}

## 10. Время и даты

- **Date format:** «1 апреля 2026» / «01.04.2026» / «April 1, 2026»
- **Relative time:** «3 дня назад» / «через 5 минут» — допустимо для context
- **Time:** «12:34» / «12:34:56» / «12:34 PM»

## 11. Конкретные примеры

### CTA-кнопки

- ✅ «Создать счёт», «Войти», «Сохранить» — короткое императивное
- ❌ «Создайте свой первый счёт прямо сейчас!» — длинно + ! + лидирующее

### Заголовки

- ✅ «Что получает бизнес», «Финансовая картина за месяц»
- ❌ «Получите финансовую картину за месяц СЕЙЧАС!»

### Error-сообщения

- ✅ «Не удалось сохранить. Проверьте подключение к интернету.»
- ❌ «Упс! Что-то пошло не так 😢 Попробуйте ещё раз!»

### Empty states

- ✅ «Нет операций за выбранный период» + кнопка «Создать первую операцию»
- ❌ «Здесь пока ничего нет... Давайте начнём! 🚀»

### Loading states

- ✅ Skeleton (без текста) или silent spinner
- ❌ «Загружаем для вас потрясающие данные...»

### Confirmation dialogs

- ✅ «Удалить операцию №142? Это действие нельзя отменить.» + Cancel/Delete
- ❌ «Вы уверены??? Точно-точно? 😅»

## 12. Inclusive language

- **Gender-neutral по умолчанию:** «пользователь» / «коллега» / «специалист»
- **Avoid assumptions:** не предполагай возраст / семейное положение / гражданство
- **A11y-первое:** alt-text для всех изображений, ARIA-labels на иконках

## 14. Length budgets

Целевая длина для типов копирайта:

| Тип | Лимит |
|---|---|
| Button label | 1-3 слова |
| Page title | 3-5 слов |
| Section heading | 2-6 слов |
| Lead paragraph | 1-2 предложения |
| Body paragraph | 3-5 предложений |
| Tooltip | 1 предложение, ≤80 chars |
| Error message | 1-2 предложения |
| Empty state body | 1-2 предложения |
| Marketing hero h1 | 4-7 слов |

- [ ] Нет запретных слов (§7)?
- [ ] Используются brand-предпочтительные слова где уместно (§8)?
- [ ] Числа в правильном формате (§9)?
- [ ] Даты в правильном формате (§10)?
- [ ] Inclusive (§12)?
- [ ] В пределах length budget (§14)?

Если хотя бы одно «нет» — переписать.
```

## Заметки

- Этот документ должен быть читаемым **за 10 минут** новым членом команды.
- Конкретные примеры (§11) — самая важная часть. Без них правила абстрактные.
- Updates через коммит с указанием reason. Voice не меняется случайно.
- Cross-check с UI: при review нового feature'а — копирайт должен пройти voice-checklist (§15).
